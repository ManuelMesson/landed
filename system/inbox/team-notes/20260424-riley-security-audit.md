## ⚠️ Doors that need attention
- `HIGH` Stored XSS risk in tracker UI. [`static/tracker.js`](/home/manuelmesson/projects/landed/static/tracker.js:16) builds HTML with `innerHTML` from API data including `job.company`, `job.role`, `analysis.role_summary`, `key_requirements`, `gaps_to_address`, `talking_points`, and profile strings. Those values are ultimately user-controlled or LLM-generated and are inserted without escaping at [`static/tracker.js`](/home/manuelmesson/projects/landed/static/tracker.js:75) and [`static/tracker.js`](/home/manuelmesson/projects/landed/static/tracker.js:91).
- `HIGH` Jordan summary/warmup UI also renders unsanitized HTML. [`static/jordan.js`](/home/manuelmesson/projects/landed/static/jordan.js:124), [`static/jordan.js`](/home/manuelmesson/projects/landed/static/jordan.js:177), [`static/jordan.js`](/home/manuelmesson/projects/landed/static/jordan.js:293), and [`static/jordan.js`](/home/manuelmesson/projects/landed/static/jordan.js:297) write API and error content directly with `innerHTML`. Since that content can contain LLM output and values derived from job posts/resume context, this is another XSS path.
- `HIGH` Generated interview audio is publicly exposed. [`backend/jordan.py`](/home/manuelmesson/projects/landed/backend/jordan.py:107) writes synthesized MP3s into `static/audio`, and [`backend/jordan.py`](/home/manuelmesson/projects/landed/backend/jordan.py:131) returns them from `/static/audio/...`. Real MP3 files are present in `static/audio/` in this workspace, so Jordan session output is currently served from a public unauthenticated path.
- `MEDIUM` `/resume-fix` skips Pydantic validation entirely. [`backend/main.py`](/home/manuelmesson/projects/landed/backend/main.py:125) accepts a raw `dict`, then trusts `.get()` values for strings and arrays. This is the only request body endpoint not using a model and makes malformed payloads and oversized prompt input easier to pass through.
- `MEDIUM` Input validation is shallow even where Pydantic is used. Models such as [`AnalyzeRequest`](/home/manuelmesson/projects/landed/backend/models.py:32), [`JobCreate`](/home/manuelmesson/projects/landed/backend/models.py:38), [`TrackResumeUpdate`](/home/manuelmesson/projects/landed/backend/models.py:77), and [`JordanRespondRequest`](/home/manuelmesson/projects/landed/backend/models.py:101) define types but no length, format, or size limits. Large resume/job-post payloads and forged client-side values can be stored as-is.
- `MEDIUM` Prompt injection surface is wide open. Raw `job_post`, `resume`, session transcript, and job excerpts are interpolated directly into Anthropic prompts in [`backend/analyzer.py`](/home/manuelmesson/projects/landed/backend/analyzer.py:107), [`backend/analyzer.py`](/home/manuelmesson/projects/landed/backend/analyzer.py:155), [`backend/jordan.py`](/home/manuelmesson/projects/landed/backend/jordan.py:196), [`backend/jordan.py`](/home/manuelmesson/projects/landed/backend/jordan.py:541), and [`backend/jordan.py`](/home/manuelmesson/projects/landed/backend/jordan.py:577) without delimiter hardening or instruction stripping.
- `MEDIUM` Auth is intentionally absent, but every data endpoint is currently public and mutable. Publicly accessible API routes include `/tracks`, `/tracks/{track_id}/resume`, `/jobs`, `/jobs/{job_id}`, `/analyze`, `/resume-fix`, `/jordan/profile/{mode}/{context_id}`, `/jordan/session/start`, and `/jordan/session/respond` in [`backend/main.py`](/home/manuelmesson/projects/landed/backend/main.py:56). Today any caller can read and modify resume, job, and coaching data.
- `LOW` Dependency audit could not be completed from this environment. `backend/requirements.txt` exists, but `cd backend && pip list --outdated` failed because `pip` is not on `PATH`, and `../.venv/bin/python -m pip ...` failed because the venv lacks the `pip` module. `pip-audit` is also unavailable here, so package freshness/CVE status remains unverified.
- `LOW` CORS is fully open in [`backend/main.py`](/home/manuelmesson/projects/landed/backend/main.py:46) with `allow_origins=["*"]` and `allow_methods=["*"]`. That is acceptable for a local prototype, but it should be tightened before any shared deployment.

## 🔒 Not applicable yet (pre-auth)
- Full authentication and authorization do not exist yet by design. There is no user identity, session management, RBAC, or ownership model.
- Once auth lands, protect all write endpoints first: `/tracks/{track_id}/resume`, `/jobs`, `/jobs/{job_id}`, `/resume-fix`, `/jordan/session/start`, and `/jordan/session/respond`.
- Once auth lands, read access should also be scoped: `/jobs`, `/jobs/{job_id}`, `/tracks`, and `/jordan/profile/{mode}/{context_id}` currently return user-specific application and coaching data with no ownership checks.
- Rate limiting and abuse controls will matter once auth or public access exists for LLM-backed endpoints, especially `/analyze`, `/resume-fix`, `/jordan/session/start`, and `/jordan/session/respond`.

## Recommended next actions
1. Replace `innerHTML` rendering of API/LLM content in `static/tracker.js` and `static/jordan.js` with safe DOM construction or strict escaping.
2. Stop serving generated Jordan audio from `/static/audio`; move it to private storage or return short-lived authenticated URLs, and clear existing generated MP3s from deployed hosts.
3. Add a `ResumeFixRequest` Pydantic model and apply length/shape limits across all request models, especially resume text, job post text, notes, and Jordan answers.
4. Add server-side guardrails for LLM prompts: hard delimiters, explicit refusal of in-band instructions from job posts, and truncation limits for pasted content and transcript history.
5. Land auth before any broader release and require ownership checks on all resume, job, and Jordan endpoints.
6. Restore dependency tooling in the project environment so `pip list --outdated` and `pip-audit` can run in CI and locally; then pin and review package versions.
7. Tighten CORS to known frontend origins once deployment moves beyond local-only testing.
